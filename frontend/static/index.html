Ecosystem Documentation & Architecture

1. High-Performance EEC Swarm Fragmentation

When a user uploads a file through the portal or S3 endpoint, it is intercepted by the Swarm Coordination Plane. The payload is processed using Reed-Solomon Erasure Coding, splitting it into $N$ data shards and $M$ parity shards (default config is 6 data + 3 parity). This allows up to 3 individual multi-cloud nodes (e.g. Dropbox or Google Drive) to go offline entirely, without causing a single byte of data loss.

2. Rest APIs Specifications & Backwards Compatibility

The coordination plane exposes rich endpoints mapped dynamically in tachyon/api/router.py. All external clients, including blockchain agents and our main S3-compatibility API routing, query these endpoints to perform parallel reads and writes.
